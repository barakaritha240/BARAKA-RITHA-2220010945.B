    • Tables

      1. The airline TABLE

        2. T, the "airline" table would store information related to airlines that operate flights.


      2. Flights TABLE.

        3. : The "flights" table keeps track of various details about each flight, including its unique ID, the airline operating it, the ticket price, and the number of available seats. This information helps manage and book flights efficiently within the flight booking system.

      3. is PAYMENT TABLE

        4. : The "payment" table helps track the financial aspect of flight bookings, including details about each payment transaction, the associated booking, payment amount, payment date, and payment method. This information is crucial for managing payments, tracking revenue, and ensuring financial transparency within the flight booking system.


      4. is passenger TABLE

        5. : The "passenger" table effectively stores information about each passenger, including their unique ID, the ID of the flight they're booked on, their name, gender, and email address. This information helps manage and track passenger details within the flight booking system.


      5. Reservation TABLE

        6. The "Reservation" table effectively stores information about each reservation, including the reservation ID, passenger ID, reservation time, and reservation date. This information helps manage and track flight reservations within the flight booking system.


      6. Admin Table

      Admin table database structure This is table is table that will be only created by admin and will hold other system users apart from admin will give them email and password that they will use to login and access the project or system, and also t used to register if she/he’s don’t have an account so once you must to create account in order to login and access or use the system. Note: the created Admin will not have same ability and right in the system.
